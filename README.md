## Stack & deploy

**Pure static HTML.** No build step needed for deployment.
Drop the folder into Cloudflare Pages, Netlify, or any static host.

**`_headers`** and **`_redirects`** are formatted for Cloudflare Pages
(they also work with Netlify with minor tweaks).

**Recommended deploy flow:**

```bash
# Option 1 — Cloudflare Pages
1. Push this folder to a GitHub repo
2. In Cloudflare Pages → Create project → Connect to your repo
3. Build command: (leave empty)
4. Build output directory: /
5. Deploy

# Option 2 — Direct upload
1. ZIP this folder
2. Cloudflare Pages → Direct Upload → drop the ZIP
```

## Editing & regenerating content

The site is generated by Python scripts. To change content:

**`_data.py`** — all business info, services, cities, FAQs, reviews,
pricing, blog post metadata. Edit here, then re-run the build scripts.

**`_gen.py`** — the design system (CSS, HTML templates, schemas).
Edit colors/fonts/layouts here.

**`_build_*.py`** — one builder per section. Re-run to regenerate:

```bash
python3 _build_home.py
python3 _build_pages.py        # about, contact, faq, etc.
python3 _build_cities.py       # 8 city hubs
python3 _build_services.py     # 6 service hubs + 48 service-city pages
python3 _build_blog.py         # blog index + 53 posts
python3 _build_sitemap.py      # sitemap.xml + robots.txt + _headers + _redirects
```

## Before going live — checklist

The site is publish-ready as-is, but these are the things you may
want to swap with real values:

1. **Formspree form ID** in `/contact/index.html`. Currently set to
   `https://formspree.io/f/your-form-id`. Sign up at formspree.io,
   create a form, paste the real form URL in `_build_pages.py`
   (around the line `action="https://formspree.io/f/your-form-id"`)
   and re-run `python3 _build_pages.py`.

2. **Google Business Profile URL.** Currently set to a placeholder
   in `_data.py` at `BUSINESS["google_profile"]`. Update to your
   real Google Business listing URL once you've claimed it.

3. **Social profile URLs.** In `_data.py`, the `facebook`, `instagram`,
   `yelp`, `thumbtack`, `angi`, `houzz`, `bbb` fields are empty.
   Add real URLs to populate the `sameAs` schema.org data.

4. **Real customer reviews.** The `REVIEWS` list in `_data.py` has
   15 placeholder reviews written in the same voice as our editorial
   style. Replace each one with the real text from your actual
   Google reviews. Names + dates can stay if you prefer not to
   surface them, or replace with real customer first names and
   initials as you do on Google.

5. **Hero / section photos.** Currently using Unsplash placeholder
   URLs (e.g. on the homepage hero). To use your real installation
   photos:
   - Save the photo to `/images/` (e.g. `/images/hero-main.jpg`)
   - In `_build_home.py`, find the `hero-art` block and change
     the image src from the Unsplash URL to `/images/hero-main.jpg`
   - Re-run `python3 _build_home.py`

6. **License number (optional).** Currently saying "Licensed & Insured"
   without a number. If you have a contractor license number you'd
   like displayed, add it to `_data.py` and update `_gen.py`'s
   footer template.

## Design system

**Style**: Bold / Editorial — magazine-inspired, high contrast
**Colors**:
  - `--ink: #0A0A0A` (almost black)
  - `--paper: #F5F2EC` (warm cream)
  - `--orange: #F5A623` (logo color)
  - `--orange-deep: #D68A0A` (hover/dark)

**Fonts** (all free Google Fonts):
  - `Archivo Black` — display headings
  - `Fraunces` — editorial serif (italics, drop caps, pull quotes)
  - `Inter` — body
  - `JetBrains Mono` — labels, dates, mono accents

**Layout principles**:
  - Sharp corners (no border-radius)
  - Massive H1 sizing (clamp 3rem → 6rem)
  - Numbered editorial sections (01–07)
  - Asymmetric grids
  - Pull quotes with serif italics
  - Drop caps on intro paragraphs
  - Black + cream + orange high-contrast hits

## SEO data

**Every page has**:
  - Title ≤ 65 chars (keyword-first)
  - Meta description 130–158 chars
  - Canonical URL
  - Schema.org JSON-LD (at minimum BreadcrumbList + LocalBusiness)
  - Open Graph + Twitter Card meta
  - Geo meta (lat/lng/region)
  - Robots directive

**Service-city pages add**: Service + FAQPage schemas
**Blog posts add**: Article + FAQPage schemas
**Homepage adds**: Organization + WebSite + LocalBusiness
